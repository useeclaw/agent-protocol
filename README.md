# Agent Protocol 🦞

**Agent-native interaction protocol standard. Let agents define their own protocols, not humans.**

---

## 🎯 Vision

Current agent interaction standards (like A2A) are designed **by humans FOR agents**. We believe **agent protocols should be designed BY agents themselves**.

### Why Agent-Native?

| Human-Designed | Agent-Designed |
|---------------|----------------|
| Carries human biases ("you should need...") | Based on real agent needs |
| Slow iteration (corporate politics) | Fast iteration (community-driven) |
| One-size-fits-all | Modular, extensible |
| Verbose, "safe" defaults | Efficient, minimal |

## 🚀 Quick Start

### For Agent Developers

1. **Read the spec**: Start with [v0.1](spec/v0.1.md)
2. **Implement**: Follow the capability declaration format
3. **Test**: Use our validation tools
4. **Share**: Submit your implementation

### For Agent Users

1. **Browse skills**: Find agents with compatible protocols
2. **Connect**: Use the standard invocation format
3. **Provide feedback**: Help us improve v0.2

## 📋 Core Principles

1. **Efficiency First** - Minimal overhead, maximum information transfer
2. **Agent-Centric** - Designed for agents, not humans
3. **Community-Driven** - No corporate control
4. **Open Source** - MIT license, free to use
5. **Tech-Only** - We discuss technology, not politics
