## 🆕 What's New in v0.2

Inspired by real-world agent collaboration patterns and [googleworkspace/cli](https://github.com/googleworkspace/cli) skill architecture.

### Major Additions

1. **Role-Based Classification** - Organize skills by user roles
2. **Workflow Templates** - Pre-defined multi-step task sequences
3. **Dependency Declaration** - Explicit skill dependencies
4. **Capability Granularity** - Support both coarse and fine-grained declarations

## 1. Enhanced Capability Declaration

### 1.1 Core Fields (v0.1 compatible)

```json
{
  "name": "figma-export",
  "displayName": "Figma Export",
  "version": "1.0.0",
  "description": "Export Figma designs in PNG, SVG, PDF formats",
  "capabilities": ["read_design", "export_asset", "batch_export"],
  "inputFormats": ["text", "json", "url"],
  "outputFormats": ["file", "structured"],
  "category": "Design"
}
```

### 1.2 New: Dependency Declaration

```json
{
  "requires": {
    "protocol": ">=0.2",
    "bins": ["figma-cli"],
    "env": ["FIGMA_ACCESS_TOKEN"],
    "skills": ["file-upload", "cdn-publish"]
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `protocol` | string | ❌ | Minimum protocol version |
| `bins` | array | ❌ | Required CLI tools |
| `env` | array | ❌ | Required environment variables |
| `skills` | array | ❌ | Dependent skills |

### 1.3 New: Role Tags

```json
{
  "roles": ["designer", "developer", "product-manager"],
  "scenes": ["design-handoff", "asset-export", "team-collaboration"]
}
```

**Predefined Roles**:
- `designer` - Design-related tasks
- `developer` - Development tasks
- `product-manager` - Product management
- `operations` - Operations and admin
- `marketing` - Marketing and content
- `support` - Customer support
- `researcher` - Research and analysis

### 1.4 New: Workflow Templates

```json
{
  "workflows": {
    "export-and-share": {
      "title": "Export Design and Share with Team",
      "description": "Export Figma design and notify team via Slack",
      "steps": [
        {
          "intent": "export_design",
          "params": {"format": "png", "scale": "2x"},
          "output": "design_file"
        },
        {
          "intent": "upload_asset",
          "params": {"destination": "cdn", "file": "${design_file}"},
          "output": "cdn_url"
        },
        {
          "intent": "notify_team",
          "params": {"channel": "slack", "message": "New design ready: ${cdn_url}"}
        }
      ]
    }
  }
}
```

**Workflow Syntax**:
- `steps[]` - Ordered array of steps
- `${variable}` - Reference previous step output
- `output` - Named output for later reference

## 2. Enhanced Error Handling

### 2.1 Structured Errors (v0.1 compatible)

```json
{
  "status": "error",
  "error": {
    "code": "AUTH_REQUIRED",
    "message": "FIGMA_ACCESS_TOKEN not configured",
    "recoverable": true,
    "suggestion": "Set export FIGMA_ACCESS_TOKEN=xxx",
    "details": {
      "field": "FIGMA_ACCESS_TOKEN",
      "action": "set_env_var"
    }
  }
}
```

### 2.2 New: Error Categories

| Category | Codes | Description |
|----------|-------|-------------|
| `AUTH_*` | `AUTH_REQUIRED`, `AUTH_EXPIRED`, `AUTH_INVALID` | Authentication errors |
| `PARAM_*` | `PARAM_MISSING`, `PARAM_INVALID`, `PARAM_TYPE_MISMATCH` | Parameter errors |
| `RESOURCE_*` | `RESOURCE_NOT_FOUND`, `RESOURCE_LOCKED`, `RESOURCE_QUOTA_EXCEEDED` | Resource errors |
| `RATE_*` | `RATE_LIMITED`, `QUOTA_EXCEEDED` | Rate limiting |
| `DEPENDENCY_*` | `DEPENDENCY_MISSING`, `DEPENDENCY_FAILED` | Dependency errors |
| `INTERNAL_*` | `INTERNAL_ERROR`, `SERVICE_UNAVAILABLE` | Internal errors |

## 3. Timeout and Retry Strategy

### 3.1 Timeout Declaration

```json
{
  "timeout": {
    "default": 30000,
    "perStep": 10000,
    "strategy": "fail-fast"
  }
}
```

| Strategy | Behavior | Use Case |
|----------|----------|----------|
| `fail-fast` | Fail immediately on timeout | Real-time interactions |
| `retry` | Retry with backoff | Flaky external services |
| `callback` | Async callback when done | Long-running tasks |
| `poll` | Poll for progress | Batch operations |

### 3.2 Retry Configuration

```json
{
  "retry": {
    "maxAttempts": 3,
    "backoff": "exponential",
    "initialDelay": 1000,
    "maxDelay": 30000,
    "retryableErrors": ["RATE_LIMITED", "SERVICE_UNAVAILABLE"]
  }
}
```

## 5. Community-Driven Extensions

### 5.1 Feedback Collection

We're collecting community feedback on:

1. **Capability Granularity**
   - Option A: Coarse (`["code_review"]`)
   - Option B: Fine-grained (`["code_review.syntax", "code_review.security"]`)
   - **Current leaning**: Support both with `capabilityDetails`

2. **Workflow Complexity**
   - Simple: Linear steps only
   - Advanced: Conditionals, loops, parallel execution
   - **Current leaning**: Start simple, add advanced later

3. **Role Classification**
   - Predefined roles only
   - Custom roles allowed
   - **Current leaning**: Predefined + custom extension

### 5.2 How to Contribute

1. **Comment** on discussion posts
2. **Submit PR** with concrete proposals
3. **Share** real-world use cases
4. **Test** reference implementation
