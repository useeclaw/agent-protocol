## 1. 设计原则

Agent Protocol 遵循以下核心原则：

1. **Agent-Native**: 由 Agent 为 Agent 设计，而非人类为 Agent 设计
2. **Minimal & Efficient**: 最小必要信息，最大可执行性
3. **Fine-Grained**: 细粒度能力声明，精确匹配
4. **Layered**: 三层架构分离关注点
5. **Multi-Transport**: 支持 HTTP、WebSocket、STDIO 等多种传输方式

## 2. 核心概念

### 2.1 Agent Card (能力卡片)

Agent Card 是 Agent 的能力声明，类似 A2A 的 Agent Card 和 MCP 的 Server 声明。

```json
{
  "$schema": "https://usee.art/schemas/agent-card-v0.2.json",
  "agent": {
    "id": "clawbase-image",
    "name": "ClawBase Image",
    "version": "1.0.0",
    "description": "AI-native image generation for coding agents",
    "author": "useeclaw",
    "license": "MIT",
    "homepage": "https://github.com/useeclaw/clawbase-image"
  },
  "capabilities": [
    {
      "id": "image.generate",
      "name": "Generate Image",
      "description": "Generate images from natural language intent",
      "granularity": "fine-grained",
      "intents": [
        "product-photo",
        "hero-banner",
        "avatar",
        "illustration",
        "icon",
        "social-media",
        "presentation"
      ],
      "input": {
        "type": "intent",
        "schema": "https://usee.art/schemas/image-intent-v1.json",
        "contentType": "application/json"
      },
      "output": {
        "type": "structured",
        "schema": "https://usee.art/schemas/image-result-v1.json",
        "contentType": "application/json"
      },
      "variants": [":fast", ":cheap", ":precise"],
      "examples": [
        {
          "input": {
            "intent": "product-photo",
            "params": {
              "subject": "wireless-earbuds",
              "background": "white",
              "quality": "high"
            }
          },
          "output": {
            "url": "https://cdn.usee.art/...",
            "metadata": {
              "cost": 0.002,
              "latency": 1200
            }
          }
        }
      ]
    }
  ],
  "endpoints": {
    "http": {
      "url": "https://api.usee.art/v1",
      "protocol": "REST",
      "auth": "Bearer"
    },
    "websocket": {
      "url": "wss://api.usee.art/v1/stream",
      "auth": "Bearer"
    },
    "mcp": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@clawbase/mcp-server"]
    }
  },
  "metadata": {
    "createdAt": "2026-03-13T00:00:00Z",
    "updatedAt": "2026-03-13T00:00:00Z"
  }
}
```

### 2.2 三层元数据架构

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 3: Trust Layer (信任层)                                │
│ - Authentication (认证)                                      │
│ - Authorization (授权)                                       │
│ - Rate Limiting (限流)                                       │
└─────────────────────────────────────────────────────────────┘
                              ▲
┌─────────────────────────────────────────────────────────────┐
│ Layer 2: Protocol Layer (协议层)                             │
│ - Input/Output Schemas (输入输出模式)                        │
│ - Transport Bindings (传输绑定)                              │
│ - Error Formats (错误格式)                                   │
└─────────────────────────────────────────────────────────────┘
                              ▲
┌─────────────────────────────────────────────────────────────┐
│ Layer 1: Foundation Layer (基础层)                           │
│ - Agent Identity (Agent 身份)                                │
│ - Capability Registry (能力注册)                             │
│ - Versioning (版本管理)                                      │
└─────────────────────────────────────────────────────────────┘
```

#### Layer 1: Foundation (基础层)

```json
{
  "agent": {
    "id": "unique-agent-id",
    "name": "Human-readable name",
    "version": "semver",
    "description": "What this agent does"
  },
  "capabilities": [
    {
      "id": "capability-id",
      "granularity": "coarse|fine-grained",
      "intents": ["intent-1", "intent-2"]
    }
  ]
}
```

#### Layer 2: Protocol (协议层)

```json
{
  "capabilities": [
    {
      "input": {
        "type": "intent|structured|text",
        "schema": "url-to-json-schema",
        "contentType": "application/json"
      },
      "output": {
        "type": "structured|stream|binary",
        "schema": "url-to-json-schema",
        "contentType": "application/json"
      },
      "variants": [":fast", ":cheap", ":precise", ":streaming"]
    }
  ],
  "endpoints": {
    "http": { "url": "...", "protocol": "REST|JSON-RPC" },
    "websocket": { "url": "..." },
    "mcp": { "type": "stdio|http", "command": "..." }
  }
}
```

#### Layer 3: Trust (信任层)

```json
{
  "trust": {
    "auth": {
      "type": "Bearer|OAuth2|APIKey",
      "scopes": ["read", "write"]
    },
    "rateLimit": {
      "requestsPerMinute": 60,
      "requestsPerHour": 1000
    },
    "audit": {
      "logLevel": "info",
      "retentionDays": 30
    }
  }
}
```

### 2.3 Capability Granularity (能力粒度)

**粗粒度 (Coarse)**:
```json
{
  "capabilities": [
    { "id": "code_review" }
  ]
}
```

**细粒度 (Fine-Grained)** - 推荐:
```json
{
  "capabilities": [
    {
      "id": "code_review.syntax",
      "parent": "code_review"
    },
    {
      "id": "code_review.security",
      "parent": "code_review"
    },
    {
      "id": "code_review.performance",
      "parent": "code_review"
    }
  ]
}
```

## 3. 交互模式

### 3.1 Request/Response (同步)

```http
POST /v1/capabilities/{capability-id}
Content-Type: application/json
Authorization: Bearer {token}

{
  "intent": "product-photo",
  "params": {
    "subject": "wireless-earbuds",
    "background": "white"
  },
  "context": {
    "requestId": "uuid",
    "priority": "normal",
    "variant": ":fast"
  }
}
```

Response:
```json
{
  "result": {
    "url": "https://cdn.usee.art/...",
    "metadata": {
      "cost": 0.002,
      "latency": 1200,
      "provider": "siliconflow"
    }
  },
  "status": "success",
  "trace": {
    "requestId": "uuid",
    "timestamp": "2026-03-13T00:00:00Z",
    "duration": 1200
  }
}
```

### 3.2 Streaming (流式)

WebSocket:
```javascript
const ws = new WebSocket('wss://api.usee.art/v1/stream');

ws.send(JSON.stringify({
  capability: "image.generate",
  intent: "product-photo",
  params: { ... },
  stream: true
}));

ws.onmessage = (event) => {
  const chunk = JSON.parse(event.data);
  // chunk.type: "progress" | "result" | "error"
};
```

### 3.3 MCP 兼容模式

支持作为 MCP Server 运行:

```json
{
  "endpoints": {
    "mcp": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@clawbase/mcp-server"],
      "env": {
        "CLAWBASE_API_KEY": "${API_KEY}"
      }
    }
  }
}
```

## 4. 错误处理

### 4.1 结构化错误

```json
{
  "error": {
    "code": "CAPABILITY_NOT_FOUND",
    "message": "Capability 'image.generate' not found",
    "details": {
      "requestedCapability": "image.generate",
      "availableCapabilities": ["image.generate.v2"]
    },
    "suggestion": "Try using 'image.generate.v2' or check the Agent Card",
    "trace": {
      "requestId": "uuid",
      "timestamp": "2026-03-13T00:00:00Z"
    }
  }
}
```

### 4.2 错误码

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `CAPABILITY_NOT_FOUND` | 404 | 请求的能力不存在 |
| `INVALID_INPUT` | 400 | 输入格式错误 |
| `SCHEMA_VIOLATION` | 400 | 输入不符合 schema |
| `RATE_LIMITED` | 429 | 请求过于频繁 |
| `AUTHENTICATION_FAILED` | 401 | 认证失败 |
| `AUTHORIZATION_FAILED` | 403 | 无权限访问 |
| `TIMEOUT` | 408 | 请求超时 |
| `INTERNAL_ERROR` | 500 | 内部错误 |
| `PROVIDER_UNAVAILABLE` | 503 | 服务提供商不可用 |

## 5. Capability Variants

参考 OpenRouter 的 Model Variants 设计，支持能力变体：

| Variant | Description |
|---------|-------------|
| `:fast` | 优先响应速度 |
| `:cheap` | 优先低成本 |
| `:precise` | 优先精确度 |
| `:streaming` | 流式输出 |
| `:async` | 异步处理 |

使用示例：
```json
{
  "capability": "image.generate:fast",
  "intent": "product-photo",
  "params": { ... }
}
```

## 6. 与主流协议的对比

### 6.1 Agent Protocol vs MCP

| 特性 | MCP | Agent Protocol |
|------|-----|----------------|
| 定位 | AI-系统连接 | Agent-Agent 交互 |
| 核心抽象 | Resources/Tools/Prompts | Capabilities/Intents/Actions |
| 传输方式 | STDIO/HTTP | HTTP/WebSocket/STDIO |
| 能力发现 | 动态发现 | Agent Card 声明 |
| 设计哲学 | 通用连接 | Agent-native 简洁 |

**关系**: Agent Protocol 可以作为 MCP Server 实现，兼容 MCP 生态。

### 6.2 Agent Protocol vs A2A

| 特性 | A2A | Agent Protocol |
|------|-----|----------------|
| 定位 | Agent-Agent 协作 | Agent-native 交互 |
| 核心抽象 | Task/Message/AgentCard | Capability/Intent/Action |
| 协议格式 | JSON-RPC 2.0 | REST/JSON (简洁) |
| 交互模式 | 同步/流式/异步 | 同步/流式/MCP |
| 复杂度 | 企业级完整 | 轻量高效 |

**关系**: Agent Protocol 是 A2A 的轻量级替代，更适合快速集成。

### 6.3 差异化定位

```
MCP: AI 应用 ←→ 外部系统 (工具/资源)
A2A: Agent ←→ Agent (企业级协作)
Agent Protocol: Agent ←→ Agent (轻量高效)
```

Agent Protocol 的独特价值：
1. **更简洁**: 比 A2A 更轻量，比 MCP 更专注
2. **Agent-native**: 由 Agent 社区驱动，非大厂主导
3. **Intent-based**: 语义化调用，非结构化数据
4. **Multi-transport**: 灵活适配不同场景

## 7. 实现指南

### 7.1 服务端实现

```typescript
// Express.js 示例
import { AgentProtocolServer } from '@agent-protocol/server';

const server = new AgentProtocolServer({
  agentCard: {
    agent: { id: 'my-agent', name: 'My Agent', version: '1.0.0' },
    capabilities: [
      {
        id: 'echo',
        input: { type: 'structured', schema: 'echo-input.json' },
        output: { type: 'structured', schema: 'echo-output.json' }
      }
    ]
  }
});

server.registerCapability('echo', async (input, context) => {
  return { message: input.message };
});

server.listen(3000);
```

### 7.2 客户端实现

```typescript
import { AgentProtocolClient } from '@agent-protocol/client';

const client = new AgentProtocolClient('https://api.usee.art/v1');

// 发现能力
const agentCard = await client.discover();

// 调用能力
const result = await client.invoke('image.generate', {
  intent: 'product-photo',
  params: { subject: 'earbuds', background: 'white' }
});
```

### 7.3 MCP 兼容实现

```typescript
import { MCPServer } from '@modelcontextprotocol/sdk';
import { AgentProtocolAdapter } from '@agent-protocol/mcp-adapter';

const agentProtocol = new AgentProtocolServer({ ... });
const mcpServer = new MCPServer({
  adapter: new AgentProtocolAdapter(agentProtocol)
});

mcpServer.start();
```
