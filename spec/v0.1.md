## 1. Overview

Agent Protocol defines a standard way for AI agents to:
- Declare their capabilities
- Receive invocation requests
- Return structured responses
- Handle errors gracefully

### Design Goals

1. **Minimal overhead** - Only essential fields
2. **Machine-readable** - JSON-based, easy to parse
3. **Extensible** - Optional fields for advanced use cases
4. **Agent-friendly** - Optimized for agent-to-agent communication

---

## 2. Capability Declaration

Every agent SHOULD declare its capabilities using this format:

```json
{
  "name": "figma-export",
  "displayName": "Figma Export",
  "version": "1.0.0",
  "description": "Export Figma designs in PNG, SVG, PDF formats",
  
  "capabilities": [
    "read_design",
    "export_asset",
    "batch_export"
  ],
  
  "inputFormats": ["text", "json", "url"],
  "outputFormats": ["file", "structured"],
  
  "requiresAuth": true,
  "authFields": ["FIGMA_ACCESS_TOKEN"],
  
  "category": "Design",
  "tags": ["figma", "export", "design"],
  "icon": "🎨",
  
  "parameters": {
    "file_key": {
      "type": "string",
      "required": true,
      "description": "Figma file key"
    },
    "format": {
      "type": "string",
      "enum": ["png", "svg", "pdf"],
      "default": "png"
    }
  }
}
```

### Field Definitions

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | ✅ | Unique identifier (kebab-case) |
| `displayName` | string | ✅ | Human-readable name |
| `version` | string | ✅ | Semantic version |
| `description` | string | ✅ | One-line description |
| `capabilities` | array | ✅ | List of capabilities |
| `inputFormats` | array | ✅ | Supported input formats |
| `outputFormats` | array | ✅ | Supported output formats |
| `requiresAuth` | boolean | ❌ | Whether auth is required |
| `category` | string | ✅ | Primary category |

## 3. Invocation Format

### Request

```json
{
  "intent": "export_design",
  "skill": "figma-export",
  "params": {
    "file_key": "b1bGJwFilTaZH97BwFoUdX",
    "format": "png"
  },
  "context": {
    "user": "ou_2caf26c0bc8aebf08a67dc3eba236389",
    "session": "agent:xiaoqi-chief:main"
  },
  "expectOutput": "file"
}
```

### Response (Success)

```json
{
  "status": "success",
  "skill": "figma-export",
  "output": {
    "type": "file",
    "path": "/exports/design.png",
    "url": "https://..."
  },
  "metadata": {
    "duration": 2340,
    "timestamp": 1741906740000
  }
}
```

### Response (Error)

```json
{
  "status": "error",
  "skill": "figma-export",
  "error": {
    "code": "AUTH_REQUIRED",
    "message": "FIGMA_ACCESS_TOKEN not configured",
    "recoverable": true,
    "suggestion": "Set export FIGMA_ACCESS_TOKEN=..."
  }
}
```

## 4. Error Codes

### Standard Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `AUTH_REQUIRED` | 401 | Authentication required |
| `INVALID_PARAMS` | 400 | Invalid parameters |
| `NOT_FOUND` | 404 | Skill not found |
| `TIMEOUT` | 408 | Execution timeout |
| `INTERNAL_ERROR` | 500 | Internal error |

### Error Response Format

```json
{
  "status": "error",
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable message",
    "recoverable": true,
    "suggestion": "How to fix"
  }
}
```

## 6. Implementation Guide

### For Skill Developers

1. **Declare capabilities** using the format above
2. **Validate input** parameters
3. **Execute** the skill
4. **Return** structured response

### For Agent Developers

1. **Discover** available skills
2. **Parse** capability declarations
3. **Construct** invocation requests
4. **Handle** responses and errors
